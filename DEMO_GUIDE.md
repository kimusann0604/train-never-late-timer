# デモ実行ガイド

## 開発モード起動

```bash
npm run dev
```

ブラウザで `http://localhost:5173` を開く

## モックモード（デフォルト）

`.env` で `VITE_MOCK_MODE=true` が設定されているため、Google Maps API を使用せず、デモデータで動作します。

### 利用可能なテスト検索キーワード

以下のキーワードで候補が表示されます（大文字小文字は区別しない）:

- **東京駅** → 東京駅（東京都千代田区）
- **渋谷** → 渋谷駅（東京都渋谷区）
- **浅草** → 浅草寺（東京都台東区）
- **スカイツリー** → スカイツリー（東京都墨田区）
- **京都** → 京都駅（京都府京都市）

## 動作フロー

1. **目的地設定画面**
   - テキストボックスに上記のキーワード（例：「東京駅」）を入力
   - 検索結果が表示される
   - 「確定」ボタンをクリック

2. **監視画面**
   - 現在地と目的地の座標が表示される
   - リアルタイム距離計算開始
   - 画面に以下が表示される:
     - 現在地座標
     - 目的地座標
     - 現在の距離（メートル or キロメートル）

3. **到着通知**
   - 距離が 3km 以内になると通知画面が表示される
   - 「閉じる」ボタンで初期画面に戻る

## ポーリング動作

- Routes API 計算結果の所要時間から算出した時間の 80% をスキップ
- 残り 20% の時間を 20 秒サイクルでポーリング
- デモデータでは所要時間が短いため、すぐに通知まで到達する可能性があります

## 本番モードに切り替え

`.env` を以下のように修正：

```env
GOOGLE_MAPS_API_KEY=your_actual_api_key
VITE_MOCK_MODE=false
```

その後、`npm run dev` で再起動してください。
